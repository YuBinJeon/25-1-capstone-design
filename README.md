# capstone-design project 
---

## **✅ 프로젝트 개요**
### **1. 프로젝트 이름**
- 미정
### **2. 프로젝트 목적**
- 수영장의 **수위, 온도, 여과 상태를 원격으로 모니터링 및 관리**하는 웹 기반 시스템 개발
### **3. 기대효과**
- 기존 수동으로 문제상황을 식별해야 했던 시스템의 개편으로 효율적 관리 환경 제공

---
## **✅ 시스템 구축 환경**
💡 **Node.js + MySQL + React** 기반으로 백엔드, 프론트엔드 구축 

---

## **✅ 시스템 개요 (최소화된 구조)**
💡 **하드웨어는 기존 제품 활용 + 소프트웨어 중심 개발**  

#### **주요 기능**
✅ **균등조 수위 모니터링** → 수위 조절기에서 신호를 받아와 DB 저장  
✅ **수영장 온도 관리** → 온도 센서 데이터 저장 및 원격 제어   
✅ **웹 대시보드** → 수위, 온도, 여과 상태를 실시간 시각화  

---

## **✅ 데이터 흐름**
**1️⃣ 하드웨어 (기존 제품 활용)**
- 균등조 수위 조절기 → 일정 간격으로 수위 데이터 신호 전달  
- 수영장 온도 센서(온도계 또는 IoT 센서) → 실시간 온도 데이터 수집  

**2️⃣ 백엔드 (Node.js + MySQL)**
- REST API 또는 WebSocket을 통해 데이터 수집  
- MySQL에 수위, 온도, 여과 상태 저장  
- 이상 상태 감지 시 알림 기능 추가 가능  

**3️⃣ 프론트엔드 (React)**
- 대시보드 UI에서 수영장 상태 실시간 확인  
- 원격으로 온수 시스템 제어 (예: 목표 온도 설정)  

---

## **✅ 8주 개발 계획**

| 주차 | 목표 | 주요 작업 |
|------|------|----------|
| **1주차** | 개발계획서 작성 | 프로젝트 내용 구체화, 개발 스택 선정|
| **2주차** | 프로젝트 환경 설정 | Node.js, MySQL, React 개발 환경 구축 |
| **3주차** | 데이터베이스 설계 및 API 기초 | MySQL 테이블 생성, 기본 API 개발 |
| **4주차** | 하드웨어 연동 테스트 | 수위 조절기 데이터 수집 방식 테스트 (시리얼, API 등) |
| **5주차** | 백엔드 API 완성 | 수위, 온도 데이터 저장 및 조회 API 구축 |
| **6주차** | 프론트엔드 기본 UI 구현 | React 대시보드 디자인 및 데이터 연동 |
| **7주차** | 원격 제어 기능 개발 | 온수 시스템 및 경고 알림 기능 추가 |
| **8주차** | 통합 테스트 및 버그 수정 | 시스템 전반 테스트 및 최적화 |
| **9주차** | 최종 배포 및 문서 작성 | 프로젝트 배포, 사용 매뉴얼 작성 |
